## Primary Operation

### CATSO — Indie Social Constellation

**CATSO** is an experimental social platform designed as an alternative to standardized, algorithm-dominated social networks.

Instead of treating profiles as interchangeable feeds, CATSO treats every profile as a **personal world** connected to a broader social constellation.

The platform includes:

* Customizable personal worlds
* Journals, galleries, shrines, and guestbooks
* Neighborhoods and community spaces
* Webrings and social constellations
* Public and private rooms
* Events, cafés, discovery systems, and editorial curation
* Collaborative world editing
* Privacy-aware media infrastructure
* Moderation, rate limiting, and row-level security
* Deterministic visual systems and interactive environments

## Technical Doctrine

I approach engineering as a systems discipline rather than a collection of isolated features.

My priorities are:

1. **Architecture before accumulation**
   Features should reinforce the system instead of creating permanent structural debt.

2. **Security as a default property**
   Authorization, isolation, validation, and data ownership must be designed into the architecture.

3. **Deterministic behavior**
   Critical interfaces and visual systems should remain predictable, testable, and reproducible.

4. **Explicit contracts**
   Boundaries between public, private, authenticated, editorial, and administrative functionality must be precise.

5. **Operational realism**
   A system is not complete because it compiles. It must survive migrations, failures, hostile inputs, maintenance, and future developers.

6. **Identity matters**
   Strong software should have a recognizable philosophy, visual language, and reason to exist.
